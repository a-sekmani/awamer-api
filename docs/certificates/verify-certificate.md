# Verify Certificate ŌĆö Backend Spec (awamer-api)

> **Module:** `CertificatesModule`
> **Endpoint:** `GET /api/v1/certificates/verify/:code`
> **Decorator:** `@Public()`
> **Rate limit:** `@Throttle({ default: { limit: 30, ttl: 60000 } })` ŌĆö 30 req/min/IP
> **Status code:** `200 OK`

---

## 1. Summary

Public endpoint that verifies a certificate by its code and
returns the holder name and the subject (path or course) the
certificate covers. Used by the "Verify at awamer.com/verify/X"
QR code printed on every certificate PDF.

The endpoint is `@Public()` ŌĆö unauthenticated callers can use it
ŌĆö but it carries a **tighter throttle** than the global default
(30/min vs 100/min) to slow hostile scanning bots without
impacting legitimate employer verification traffic. See comment
on `verify()` in `certificates.controller.ts` (C2 resolution).

---

## 2. Request

```
GET /api/v1/certificates/verify/:code
```

`:code` is the certificate's `certificateCode` field ŌĆö a
12-character lowercase hex string generated by
`CertificatesService.generateCode()`. The route does **not** use
`ParseUUIDPipe`; codes are not UUIDs.

---

## 3. Behavior ŌĆö `CertificatesService.verifyByCode(code)`

Source: `src/certificates/certificates.service.ts` `verifyByCode()`.

1. **Unique lookup** on `certificateCode`:
   ```ts
   prisma.certificate.findUnique({
     where: { certificateCode: code },
     include: {
       user:   { select: { name: true } },
       path:   { select: { title: true, slug: true } },
       course: { select: { title: true, slug: true } },
     },
   });
   ```
2. **Missing ŌåÆ `NotFoundException('Certificate not found')`**.
   This is the **only** way an unknown code is reported ŌĆö there
   is no rate-limit leak, no timing side channel, no "valid but
   expired" state. Certificates do not expire.
3. **Build `subject`** from either the `path` or `course`
   relation based on `cert.type`:
   ```ts
   const subject = cert.type === 'PATH'
     ? { type: 'PATH',   title: cert.path!.title,   slug: cert.path!.slug   }
     : { type: 'COURSE', title: cert.course!.title, slug: cert.course!.slug };
   ```
4. **Return** a verification DTO:
   ```ts
   {
     valid: true,
     type: cert.type,
     issuedAt: cert.issuedAt.toISOString(),
     holder: { fullName: cert.user.name },
     subject,
   }
   ```

---

## 4. Successful response

```json
{
  "data": {
    "valid": true,
    "type": "PATH",
    "issuedAt": "2026-04-10T12:00:00.000Z",
    "holder": { "fullName": "Ahmad Sekmani" },
    "subject": { "type": "PATH", "title": "Full-Stack Path", "slug": "full-stack-path" }
  },
  "message": "Success"
}
```

The `valid: true` flag is always `true` in the success branch ŌĆö
the endpoint has no "valid but warning" state. An invalid code is
a 404, not a `valid: false` body.

---

## 5. Error responses

| Status | When |
|--------|------|
| `404`  | `Certificate not found` ŌĆö no row with the given code. |
| `429 RATE_LIMIT_EXCEEDED` | Bot scanning ŌĆö 30 req/min/IP tripped. |
| `500 INTERNAL_ERROR` | Prisma lookup failed. |

No 400: the `:code` path parameter is a string, no format
validation. An odd code shape (e.g. length 6) simply returns 404.

No 401 or 403: the endpoint is public.

---

## 6. Privacy considerations

The response carries the holder's **full name** and the subject
title/slug. It does **not** carry:

- The user id.
- The user's email.
- Any progress or enrollment detail.
- Any other certificates held by the same user.

That is the minimum needed to satisfy "prove this person earned
this credential", and the verification surface has no other
information to leak.

If the learner wants to hide their name from verification, they
would need to delete the certificate ŌĆö no "hide" state exists in
the schema.

---

## 7. Throttle rationale

`@Throttle({ default: { limit: 30, ttl: 60000 } })` overrides the
global default of 100/min. The choice:

- **Higher than auth routes** (5/min on login) because legitimate
  employer verification is a sparse but real use case.
- **Lower than the global** so a scraper scanning 100 codes/sec
  hits the limit within 1 IP-second.

The comment in the controller source tags this as clarification
C2.

## 11. Things NOT to change without coordination

- The `@Public()` decorator. Employers must be able to verify
  without logging in.
- The 30/60000 throttle. The QR-code verification UX relies on
  this specific value.
- The "404 for unknown code, no details" behavior. Returning
  `valid: false` would confirm the code format is recognized,
  leaking information.
- The fields in the response. Adding the holder's email or
  user id would break the privacy minimum.
